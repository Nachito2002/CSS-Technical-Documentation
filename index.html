Efecto cascada

Cuando damos estilo a un elemento HTML, este se aplicará al elemento indicado pero también será heredado por su contenido, a no ser que otra regla o selector sobreescriba el estilo para alguno de los hijos.

El estilo se puede dar inline mediante el atributo style o con selectores, los cuales se pueden colocar dentro de la etiqueta style o en una hoja CSS referenciada dentro del head.

Selectores

Un selector es un conjunto de reglas de estilos aplicadas múltiples elementos de la hoja html. En lugar de aplicar el estilo elemento por elemento, lo aplicamos de manera global.

Principales tipos de selectores

Id

Se define mediante una # justo antes del nombre de la id. Las reglas de estilo que contiene se aplican al elemento que tenga la id indicada.

#id{
    /* Estilos */
}

Clase

Se define mediante un . justo antes del nombre de la clase. Las reglas de estilo que contiene se aplican a cualquier elemento que tenga la clase indicada. Debido a que un elemento puede tener varias clases, también puede recibir estilo de varios selectores de clase. 

.clase{
    /* Estilos */
}

Elemento

Aplica estilo a todos los elementos que tengan la etiqueta html indicada. Por ejemplo, un selector del elemento div aplicaría estilos a todos los div.

div{
    /* Estilos */
}
  • En caso de contradicción entre selectores, se aplicarán esta jerarquía de prioridad: !important > inline style > ids > clases > elementos.
  • En caso de igual importancia entre selectores que se contradicen, los últimos en haber sido escritos tendrán prioridad.
  • La aplicación de estilos inline se penaliza en SEO.

Variables

Declaración y asignación: --nombrevariable: value;

Utilización: propiedad: var(--nombrevariable, fallback);

  • El fallback no es necesario. Se utiliza para que el navegador tenga un respaldo en caso de que la variable sea inválida. Es útil para debuggear.
  • Otra manera de hacer respaldos es poner la misma propiedad justo antes con un valor con más soporte (aprovechando que, en caso de igual importancia, la prioridad va de abajo hacia arriba).

Las variables en CSS también tienen efecto cascada. Por tanto, son utilizables dentro del elemento en el que han sido creadas y de cualquiera de los elementos que estén anidados dentro. Por esto, las variables CSS suelen declararse en la pseudoclase :root (contenedor de todo el documento .html).

Las variables se pueden sobreescribir volviéndolas a declarar dentro de elementos más específicos que aquel en el que fueran declaradas por primera vez.

Media queries

Permiten cambiar los valores de las reglas de un selector en función del tamaño del viewport. Se utilizan para hacer contenido adaptativo. También pueden alterar el valor de las variables.

@media (max-width: 350px) {
    :root {
    --penguin-size: 200px;
    }
    p {
    color: red;
    }
}

Esta media querie se activaría si el ancho del viewport es menor o igual a 350 píxeles.

Unidades

Unidades absolutas

  • px: pixeles.
  • %: porcentaje respecto al elemento padre.

Unidades relativas

  • em: cambia el tamaño de un elemento con un multiplicador al tamaño de la fuente de ese mismo elemento, por lo que si el tamaño de la fuente cambia, el resultado de esta unidad también. Si se cambia font-size con em, se calculará en función del tamaño de la fuente del padre.
  • vw: 10vw representan el 10% del ancho del viewport.
  • vh: 3vh representan el 3% del alto del viewport.
  • vmin: 70vmin representan el 70% de la dimensión más pequeña (ancho o alto).
  • vmax: 100vmax representan el 100% de la dimensión más grande (ancho o alto).

Se pueden poner margin y padding negativos.

En los atributos en los que se pueda cambiar el valor de los cuatro lados del elemento, los valores se introducen en sentido de las agujas del reloj empezando por top.

#RRGGBB se puede abreviar a #RGB. El navegador lo traducirá automáticamente. Por ejemplo, #FF0000 (rojo) se puede traducir a #F00.

Posiciones

  • position: relative; reposiciona el elemento respecto a su posición dentro del flujo normal. Su posición real dentro del flujo normal no cambia, por lo que otros elementos lo tratan como si estuviera en su posición por defecto.
    • Se aprovecha con las propiedades left, right, top y bottom para indicar el offset visual que tiene que tener en píxeles, porcentajes o ems.
    • Con reposicionamiento o sin él, el html tiene que mantener sentido cuando se lee de arriba a abajo por razones de SEO y para gente con visibilidad impedida.
  • position: absolute; mueve el elemento en relación a su contenedor. Al contrario que relative, esto saca al elemento del flujo normal, de manera que otros a su alrededor dejarán de respetar el espacio que ocuparía normalmente.
    • La posición será respecto al primer padre que tenga una propiedad position definida.
    • También utiliza las propiedades left, right, top y bottom para reposicionarse.
  • position: fixed; se comporta igual que absolute, pero se reposiciona en relación a la ventana del navegador. Por tanto, nunca cambia de posición dentro de esta a pesar de que el usuario haga scroll.
  • float: left/right; saca al elemento del flujo normal y lo empuja a la izquierda o la derecha dentro de su contenedor. Normalmente se usa en conjunto con width para indicar cuánto espacio horizontal debe ocupar.
  • z-index: xxx; por defecto, cuando varios elementos se superponen, la prioridad en el eje Z va de abajo hacia arriba al leer el documento. Z-index cambia a la fuerza la prioridad en el eje Z. Cuanto mayor sea el valor en z-index, mayor será su prioridad.
  • border-sizing: controla si width y height se aplican sobre el contenido o sobre todo.
    • content-box: valor por defecto. Width y height se aplican sobre el contenido.
    • border-box: width y height se aplican sobre contenido+padding+border.

Accesibilidad

Elementos visibles sólo para lectores de pantalla

Esto es útil para mantener elementos que son útiles para el lector de pantalla pero que no queremos que sean visibles para la mayoría de usuarios, como por ejemplo el h1.

.sr-only {
    position: absolute;
    left: -10000px;
    width: 1px;
    height: 1px;
    top: auto;
    overflow: hidden;
}

Ratio de contraste y daltonismo

Las WCAG recomiendan que el ratio de contraste de texto respecto a fondo sea de por lo menos 4,5:1.

Se debe evitar destacar texto exclusivamente mediante tono mientras se mantienen saturación o luminosidad similares a las del contenido que lo rodea, o diferenciar fondo de primer plano sólo por luminosidad o saturación (sin cambiar o cambiando poco el tono).

Principios de diseño receptivo

Imagenes adaptativas

img {
    transform-origin: left top;
    transform: scale(0.5);
    max-width: 100%;
    height: auto;
    display: block;
}

Botones

a:link, a:visited {
    background-color: #f44336;
    color: white;
    padding: 15px 25px;
    text-align: center;
    text-decoration: none;
    display: inline-block;
}
a:hover, a:active {
    background-color: red;
}

Texto adaptativo

En lugar de usar em o px para dar tamaño al texto, se pueden utilizar unidades de viewport (vw, vh, vmin ó vmax) para crear texto adaptativo.
